# Submission Checklist

Tick off every item before submitting.

---

## Code

- [ ] My Flutter project runs cleanly with `flutter pub get && flutter run` on a fresh machine
- [ ] `flutter analyze` shows no errors (warnings are OK)
- [ ] The screen renders on a phone size (~390 × 844) without overflow errors
- [ ] The screen renders on a desktop size (~1280 × 800) without overflow errors
- [ ] I used only the packages in `allowed_packages.md`
- [ ] The screen uses the colours / fonts / spacing from `design_tokens.md`
- [ ] No commented-out code, no `print()` debug statements

---

## Repo

- [ ] Code is in a public GitHub repo
- [ ] The repo has a `README.md` explaining:
  - Which reference image I chose (1, 2, or 3)
  - How to run the project (`flutter pub get && flutter run`)
  - Any design decisions I made where the reference was ambiguous
- [ ] The repo is **not** a clone of any other Flutter project — it's a fresh project

---

## Recording

- [ ] 1-minute screen recording of the running app, uploaded to one of:
  - Loom (recommended)
  - YouTube as unlisted
  - Google Drive shared with link access "anyone with the link can view"
- [ ] The recording shows the screen at both phone and desktop sizes
- [ ] The recording is **not** a static screenshot — it must be a moving video showing scroll / interaction if any

---

## Submission

Send one email to the address shared with you, with:

- Subject line: `Flutter UI Task Submission — [Your Name] — Screen [1/2/3]`
- GitHub repo link
- Screen recording link
- 3-line note about which design decisions you made

That's it. Hit send.

---

## Deadline

**28 May 2026, 11:59 PM IST.**

Submissions received after this time are not reviewed.

---

## If something breaks at submit time

If you genuinely cannot push to GitHub or upload a video by the deadline, send what you have — partial repo, raw video file as attachment, anything — before the deadline. We'd rather see a broken submission on time than a perfect one a day late.
